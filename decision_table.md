# デシジョンテーブル: modules/check_process.py

このデシジョンテーブルは、`modules/check_process.py` の `check_values` 関数におけるカラム型ごとの値チェック・クリーニング処理を整理したものです。

| 条件/カラム型 | 規則1 | 規則2 | 規則3 | 規則4 |
|:-------------|:------|:------|:------|:------|
| カラムがDataFrameに存在しない | Y | N | N | N |
| 型が'datetime' | N | Y | N | N |
| 型が'float'    | N | N | Y | N |
| 型が'int'      | N | N | N | Y |
| 型が'str'      | N | N | N | N |
| 動作: スキップ | X | - | - | - |
| 動作: 日付フォーマットチェック（不正値は空文字にし警告） | - | X | - | - |
| 動作: float変換チェック（不正値は空文字にし警告） | - | - | X | - |
| 動作: int変換チェック（不正値は空文字にし警告） | - | - | - | X |
| 動作: 文字列型へ変換・欠損値補完 | - | - | - | - |

## ポイント
- カラムがDataFrameに存在しない場合はスキップされます。
- 型ごとに値の検証・クリーニング処理が異なります。
- 不正値は空文字に置換され、警告リストに追加されます。
- すべてのカラムで最終的に欠損値（NaN等）は空文字で補完されます。
- column_typesに含まれないカラムは削除されます。

このテーブルは、複雑な条件分岐を視覚的に整理するためのものです。
